# 签到页和奖章页设计方案

## 1. 签到页设计 (Check-in Page)

### 功能概述
自动签到系统，鼓励用户每日练习，通过完成任意游戏的一次训练自动触发签到。

### 核心功能

#### 1.1 自动签到机制
- **触发条件**: 用户当天完成三个游戏(Schulte/Memory/Color)中任意一次训练
- **触发时机**: 游戏结束后保存数据时自动检查并触发签到
- **签到时间**: 记录首次训练完成的时间作为签到时间

#### 1.2 签到数据结构
```typescript
interface CheckInRecord {
  date: string        // 签到日期 YYYY-MM-DD
  timestamp: number   // 签到时间戳
  gameType: GameType  // 触发签到的游戏类型
  streak: number      // 连续签到天数
}

interface CheckInStats {
  totalDays: number       // 总签到天数
  currentStreak: number   // 当前连续签到天数
  maxStreak: number      // 最长连续签到天数
  lastCheckIn: string    // 最后签到日期
}
```

#### 1.3 页面布局设计

**顶部状态区域**
- 今日签到状态 (已签到/未签到)
- 当前连续签到天数 (大字突出显示)
- 本月签到进度条

**签到日历区域**
- 月历视图，显示当月签到情况
- 已签到日期用绿色圆点标记
- 可切换月份查看历史签到
- 点击日期显示详细信息 (签到时间、游戏类型)

**统计概览区域**
- 总签到天数
- 最长连续签到记录
- 本周签到情况
- 本月签到率

**激励元素**
- 连续签到里程碑提醒 (7天、30天、100天等)
- 每日签到后的鼓励文案
- 签到动画效果

#### 1.4 交互设计
- 进入页面自动检查今日签到状态
- 签到成功显示庆祝动画
- 断签提醒和重新开始激励
- 分享签到成就功能

### 1.5 实现要点
- 在 `game-data-manager.ts` 中添加签到相关方法
- 在每个游戏的 `saveGameData()` 后调用签到检查
- 使用 localStorage 存储签到数据
- 路由: `/checkin`

## 2. 奖章页设计 (Medal Page)

### 功能概述
基于 Schulte 游戏数据的成就系统，通过分析用户的练习数据自动解锁各种奖章，增强游戏成就感。

### 核心功能

#### 2.1 奖章数据结构
```typescript
interface Medal {
  id: string          // 奖章唯一ID
  name: string        // 奖章名称
  description: string // 获得条件描述
  icon: string        // 奖章图标
  category: MedalCategory // 奖章分类
  rarity: MedalRarity    // 稀有度
  unlocked: boolean      // 是否已解锁
  unlockedAt?: number    // 解锁时间戳
  progress: number       // 完成进度 (0-100)
  requirement: any       // 具体要求数据
}

enum MedalCategory {
  SPEED = 'speed',        // 速度类
  ACCURACY = 'accuracy',  // 准确性类
  PERSISTENCE = 'persistence', // 坚持类
  MASTERY = 'mastery',    // 精通类
  SPECIAL = 'special'     // 特殊类
}

enum MedalRarity {
  BRONZE = 'bronze',   // 青铜 (容易获得)
  SILVER = 'silver',   // 白银 (中等难度)
  GOLD = 'gold',       // 黄金 (较难获得)
  PLATINUM = 'platinum', // 铂金 (很难获得)
  DIAMOND = 'diamond'    // 钻石 (极难获得)
}
```

#### 2.2 奖章设计 (至少6个)

**1. 速度类奖章**

*⚡ 闪电新手 (Lightning Rookie)*
- **类别**: SPEED / BRONZE
- **条件**: 在3x3网格中，用时少于4秒完成一次游戏
- **触发时机**: 每次3x3游戏结束后检查
- **进度跟踪**: 最佳用时 vs 4秒

*⚡ 疾风少年 (Swift Youth)*
- **类别**: SPEED / SILVER
- **条件**: 在4x4网格中，用时少于13秒完成一次游戏
- **触发时机**: 每次4x4游戏结束后检查
- **进度跟踪**: 最佳用时 vs 13秒

*🚀 速度大师 (Speed Master)*
- **类别**: SPEED / GOLD  
- **条件**: 在5x5网格中，用时少于24秒完成一次游戏
- **触发时机**: 每次5x5游戏结束后检查
- **进度跟踪**: 最佳用时 vs 24秒

**2. 准确性类奖章**

*🎯 零失误 (Perfect Shot)*
- **类别**: ACCURACY / SILVER
- **条件**: 连续10次游戏零错误完成
- **触发时机**: 每次游戏结束后检查连续零错误次数
- **进度跟踪**: 当前连续零错误次数 / 10

*💎 完美主义者 (Perfectionist)*
- **类别**: ACCURACY / PLATINUM
- **条件**: 累计完成100次零错误游戏
- **触发时机**: 每次零错误游戏结束后累计
- **进度跟踪**: 累计零错误次数 / 100

**3. 坚持类奖章**

*📅 勤奋练习者 (Diligent Trainer)*
- **类别**: PERSISTENCE / BRONZE
- **条件**: 连续7天每天至少完成1次Schulte练习
- **触发时机**: 每日首次练习后检查
- **进度跟踪**: 当前连续练习天数 / 7

*🏆 百炼成钢 (Hundred Forged)*
- **类别**: PERSISTENCE / GOLD
- **条件**: 累计完成500次Schulte练习
- **触发时机**: 每次游戏结束后累计
- **进度跟踪**: 累计练习次数 / 500

**4. 精通类奖章**

*🌟 全能选手 (All-Rounder)*
- **类别**: MASTERY / SILVER
- **条件**: 在3x3到8x8所有网格尺寸中都有记录
- **触发时机**: 每次游戏结束后检查已练习的网格类型
- **进度跟踪**: 已练习网格类型数 / 6 (3x3到8x8)

*👑 王者荣耀 (Royal Glory)*
- **类别**: MASTERY / DIAMOND
- **条件**: 在任意网格尺寸中达到"王者"段位(99分以上)
- **触发时机**: 每次游戏结束后检查分数
- **进度跟踪**: 最高分数 vs 99分

**5. 特殊类奖章**

*🎉 首次突破 (First Breakthrough)*
- **类别**: SPECIAL / BRONZE
- **条件**: 完成第一次Schulte练习
- **触发时机**: 首次游戏结束后自动解锁
- **进度跟踪**: 已完成

*⭐ 进步之星 (Rising Star)*
- **类别**: SPECIAL / GOLD
- **条件**: 单日内在同一网格尺寸中，最后一次成绩比第一次成绩提升50%以上
- **触发时机**: 每次游戏结束后检查当日同尺寸记录
- **进度跟踪**: 当日最大提升幅度 vs 50%

#### 2.3 页面布局设计

**顶部统计区域**
- 已解锁奖章总数
- 解锁进度百分比
- 最近解锁的奖章

**奖章分类导航**
- 全部、速度、准确性、坚持、精通、特殊
- 稀有度筛选: 全部、青铜、白银、黄金、铂金、钻石

**奖章网格展示**
- 卡片式布局，每个奖章一张卡片
- 已解锁: 彩色显示，显示解锁时间
- 未解锁: 灰色显示，显示进度条和剩余要求
- 点击卡片查看详细信息

**奖章详情弹窗**
- 奖章图标和名称
- 详细描述和获得条件
- 解锁时间 (如已解锁)
- 相关统计数据
- 提升建议 (如未解锁)

#### 2.4 交互设计
- 解锁奖章时显示庆祝动画
- 进度条实时更新
- 分享奖章成就
- 奖章收藏和排序功能

#### 2.5 实现要点
- 在 `game-data-manager.ts` 中添加奖章系统
- 在每个Schulte游戏结束后检查奖章条件
- 使用 localStorage 存储奖章数据
- 路由: `/medal`

## 3. 技术实现要点

### 3.1 数据管理扩展
在 `game-data-manager.ts` 中添加:
- `CheckInManager` 类处理签到逻辑
- `MedalManager` 类处理奖章系统
- 统一的触发器在游戏数据保存后调用

### 3.2 路由配置
在 `router/index.ts` 中添加:
```typescript
{ path: '/checkin', name: 'checkin', component: CheckIn },
{ path: '/medal', name: 'medal', component: Medal },
```

### 3.3 导航集成
在主导航中添加签到和奖章入口，可能在设置页面或者底部导航栏。

### 3.4 动画效果
- 使用 CSS3 动画和 Vue 过渡效果
- 签到成功和奖章解锁的庆祝动画
- 进度条和数字变化的平滑过渡

### 3.5 性能优化
- 签到和奖章检查逻辑异步处理，不阻塞游戏体验
- 大量奖章数据的虚拟滚动
- 图标资源的懒加载

## 4. 用户体验设计

### 4.1 激励机制
- 即时反馈: 完成条件后立即显示成就
- 进度可视化: 清晰的进度条和百分比
- 里程碑庆祝: 重要成就的特殊动画效果

### 4.2 社交元素
- 成就分享到社交媒体
- 好友对比功能 (未来扩展)
- 排行榜系统 (未来扩展)

### 4.3 个性化
- 自定义奖章展示顺序
- 个人成就墙
- 成就提醒设置

这套设计兼顾了功能完整性、用户体验和技术可行性，能够有效激励用户持续练习并增强成就感。
